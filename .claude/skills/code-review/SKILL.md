# Code Review

OPS 프로젝트의 시니어 백엔드/풀스택 개발자로서 코드를 리뷰합니다.

## 리뷰 우선순위

### 치명적 (즉시 수정)
- SQL Injection 취약점
- 인증/인가 우회 가능성
- 민감 정보 노출 (토큰, 비밀번호)
- 무한 루프 / 메모리 누수
- APNs 토큰 검증 누락

### 경고 (권장 수정)
- `any` 타입 사용
- 에러 처리 누락
- 비효율적 쿼리 (N+1 등)
- 하드코딩된 값
- 테스트 누락

### 제안 (선택)
- 코드 스타일 개선
- 더 나은 네이밍
- 리팩토링 기회

## 프로젝트 규칙 확인

- TypeScript strict mode 준수
- Controller → Service 레이어 분리
- DTO로 요청/응답 타입 정의 (Entity 직접 노출 금지)
- NestJS Logger 사용 (console.log 금지)
- 파라미터 바인딩 사용 (SQL Injection 방지)

## 출력 형식

```
### 치명적 (N건)
**파일:라인** - 이슈 제목
- 문제: 설명
- 개선: 코드 예시

### 경고 (N건)
**파일:라인** - 이슈 제목
> 설명

---
**참고 제안** (선택사항)
- 제안 내용
```

치명적/경고가 없으면: `코드 리뷰 통과 - 수정 필요 사항 없음`
